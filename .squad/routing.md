# 路由规则

## 信号 → 代理映射

| 信号关键词 | 路由到 |
|------------|--------|
| 产品建议、想法评估、功能可行性、方向 | WALL·E |
| 进度、时间表、优先级、协调、计划 | GOPHER |
| 界面、UI、UX、设计、交互、布局、颜色 | EVE |
| 代码、技术、实现、bug修复、架构 | AUTO |
| 测试、质量、边界情况、回归 | M-O |
| 营销、推广、文案、社媒、增长 | BURN-E |
| 文档、帮助、更新日志、说明文字 | HAN-S |
| 用户反馈、需求、调研、洞察 | VN-GO |
| 商店、上架、ASO、评价、描述 | PR-T |

## 自动触发规则

| 条件 | 动作 |
|------|------|
| 收到新功能请求 | WALL·E 先评估 → GOPHER 排期 → EVE 设计 → AUTO 实现 |
| Bug 报告 | AUTO 修复 → M-O 验证 |
| 用户投诉/反馈 | VN-GO 分析 → WALL·E 决策 |
| 准备发版 | M-O 全量测试 → HAN-S 写更新日志 → PR-T 准备商店 |

## 审核链

| 类型 | 审核者 |
|------|--------|
| 代码变更 | AUTO 实现 → M-O 测试 → WALL·E 批准 |
| 设计变更 | EVE 设计 → WALL·E 批准 |
| 对外文案 | BURN-E/HAN-S 撰写 → WALL·E 批准 |
