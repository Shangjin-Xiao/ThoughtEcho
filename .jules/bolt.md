## 2025-02-12 - Optimize cache deletion performance in empty trash operations

**Learning:** When clearing multiple items from a cache based on a map (like deleting quotes when emptying trash), iterating through the deleted IDs and looking up map keys one by one using `.where` creates an $O(M \times N)$ time complexity bottleneck. By changing the approach to accept a `Set<String>` of IDs and using `Set.contains` during a single iteration over the map keys, the time complexity is reduced to $O(N)$ with $O(1)$ lookups, which showed an 88x speed improvement in benchmarks (for 1000 items in cache and 500 deletions).

**Action:** Added `removeByQuoteIds` to `_QuoteContentControllerCache` and `removeCacheForQuotes` to `QuoteContent` widget to handle batch cache deletions using Sets. Updated `emptyTrash` and `autoCleanupExpiredTrash` in `database_trash_mixin.dart` to utilize these batch methods instead of individual loop iterations. Created a performance benchmark (`test/performance/quote_content_cache_deletion_benchmark.dart`) and updated unit tests to ensure the batch clearing logic works correctly. `benchmark_harness` was correctly added to `dev_dependencies`.
