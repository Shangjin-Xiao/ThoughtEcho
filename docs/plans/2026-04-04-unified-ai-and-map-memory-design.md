# 统一 AI 容器、探索页与地图画廊 最终设计方案 (2026-04-04)

> 本文档记录了关于 Explore 中枢页、统一 AI 容器（单笔记模式 vs Agent模式）、AI Skills 技能系统以及地图回忆的最终产品交互设计。

## 1. 探索页 (Explore Page) UX 设计

维持“数据看板优先”的清晰视觉层级，彻底梳理原有“周期报告”与“洞察”的层级关系：

1. **顶部数据看板 (Data Dashboard)**
   - 展示选定周期（本月/本周/本年，选择器可置于下方）的统计数据：笔记数量、总字数、活跃天数、常见时段、常见天气、常用标签。
   - **动态引导语 (Greeting)**：根据上述数据生成的一段总结语。若未配置 AI API，显示本地拼接的文本；若已配置，则由 AI 生成。
   - *注意*：这段生成的总结语，也会作为用户从探索页进入“AI 助手”时的初始气泡/引导语，但**不作为**后续 AI 对话的历史上下文。

2. **中部核心入口 (Action Cards)**
   - 左侧：**AI 助手 (Agent 入口)**
   - 右侧：**地图回忆 (画廊入口)**

3. **底部内容列表 (Content Lists)**
   - 喜爱记录 (Favorites)
   - 最近笔记 (Recent Notes)
   - 周期选择器（月/周/年）等操作区

## 2. AI Skills (技能系统) 与底层架构

将原有的硬编码“斜杠命令”全面升级为可扩展的 **AI Skills (技能系统)**。这是连接固定工作流与自由 Agent 的桥梁。

- **技术栈选型 (Best Practice)**：不引入臃肿的 LangChain，采用**原生 HTTP (Dio) 对接 OpenAI 标准的 Tool Calling (JSON Format) 协议**，在 Flutter 端实现极简的推理引擎，确保流式 UI 状态的精确控制和极速响应。
- **技能 (Skill) 定义**：每个 Skill 包含名称、图标、触发词（如 `/润色`）、系统提示词 (System Prompt) 以及是否需要外部工具权限。
- **内置技能**：系统自带经典的三件套（如 润色文本、续写内容、深度分析、分析来源）。
- **用户自定义技能**：允许用户在设置中新建 Skill，自行编写 Prompt，并设定专属的斜杠触发词。

## 3. 大一统 AI 容器 (`AIAssistantPage`)

彻底废弃目前零散的页面，全应用收敛至唯一的 `AIAssistantPage`。严格区分以下两种模式，体验完全不同：

### 3.1 模式 A：单笔记对话模式 (沙盒环境)
- **入口**：从首页单条笔记、或全屏编辑器中的 AI 按钮进入。
- **状态**：界面顶部悬浮“当前笔记”卡片。
- **能力限制**：**无法**调用外部网络或全库搜索工具。完全聚焦于对当前这一条笔记的文本处理。
- **Skills 的显式调用 (Slash Commands)**：
  - 用户在此模式下输入 `/`，弹出所有可用 Skills 的列表（含内置与用户自定义）。
  - 选择后（如 `/润色`），系统提取当前笔记内容结合该 Skill 的 Prompt 发送给 LLM。
  - **呈现闭环**：AI 返回带有“替换原笔记”或“追加到原笔记”操作按钮的结果卡片。点击后直接带回编辑器手动保存，无缝衔接。

### 3.2 模式 B：全局 Agent 模式 (全能助理)
- **入口**：从 Explore 探索页进入（默认无笔记绑定），或从单笔记模式切换而来（保留原笔记绑定）。
- **状态**：若从探索页进入，开局显示动态引导语；若从单笔记模式切换而来，笔记仍作为上下文保留。
- **核心能力 (工具箱)**：
  - **跨笔记检索**：基于用户描述搜索过往笔记（含周期洞察：程序发送所有笔记，或 Agent 自主搜索重新查看某条）。
  - **网络访问**：联网搜索、直接读取网页内容。
  - **跨端操作**：例如搜索到诗句后，卡片上提供【保存为新笔记】的按钮。
- **Skills 的隐式触发 (Auto-Routing)**：
  - 不需要死板的斜杠命令。用户直接用自然语言吩咐。
  - Agent LLM 会将用户库中的所有 Skills 作为“可用工具 (Tools)”来理解，当意图匹配时自动调用。

### 3.3 模式无缝切换与会话管理
- **升级为 Agent 模式**：在“单笔记模式”下，可一键开启“Agent 模式”开关。
  - **弹窗确认**：首次开启时弹出安全确认框：“开启 Agent 模式后，AI 将具备调用外部工具（如联网、全库搜索）的能力”，并提供“不再提醒”勾选项。
  - **保留上下文 (不中断对话)**：切换为 Agent 模式后，**当前绑定的笔记不会被解除**，依然作为强大的上下文存在。不仅不会重新开启对话，之前的聊天记录也完全保留。
- **降级为单笔记模式**：若在无笔记绑定的 Agent 模式下想切回单笔记模式，系统会提示“请先从记录页选择一条具体笔记进入”。
- **新开对话 (New Chat)**：AI 页面顶部导航栏提供明确的“开启新对话”按钮。点击后清空当前屏幕内容，开启全新的持久化会话（从单笔记重新开始或全新的全局 Agent 会话）。

## 4. 地图回忆全景 (`MapMemoryPage`)

区分“打卡定位工具”和“回忆画廊”的职责。

- **定位打卡 (`MapLocationPickerPage`)**：专供全屏编辑器使用（长按定位键呼出，类似微信朋友圈选地点），用于给单条笔记打上精确的 `poiName` 和经纬度。
- **回忆画廊 (`MapMemoryPage`)**：
  - 从 Explore 页进入的纯展示页面。
  - **初始视角与空状态**：进入后默认呈现**全局总览视角**，让用户一眼看到所有笔记分布的大致轮廓。若当前没有任何带经纬度的笔记，依然展示真实的地图（定位到当前位置附近），并在地图上方悬浮气泡提示：“目前还没有带坐标的笔记”。
  - **海量数据聚合 (Clustering)**：**永远展示全部足迹**，不做时间或标签过滤。通过地图的聚合算法 (Cluster)，缩小地图时聚合为数字气泡，放大地图时像手机相册一样平滑展开具体的笔记标记，享受“点亮世界”的成就感。
  - **无缝点击浏览**：点击地图上的具体标记，从屏幕底部滑出一个半屏面板 (Bottom Sheet)，面板内直接复用 `QuoteItemWidget` 展示该笔记的完整内容和标签。点击该卡片可深度进入全屏编辑器，向下滑动则快速关闭面板继续游览。
